Git

Adicionando / Removendo alterações no stage

Stage é o local onde as alterações vão ficar prontas para do commit

git add . Adiciona todas as alterações ao stage. Deixando pronto para fazer commit

git add nomeArquivo Adiciona apenas um arquivo ao stage

git add *.js Adiciona todos os arquivos com extensão "js"

git reset -- nomeArquivo Remove o arquivo do stage e se não especificar o arquivo, remove todos

git add -i Abre o menu de interação que possibilita:

  1. Escolha a opção "revert", enter, e depois o número do arquivo que quer tirar do stage, depois aperte enter. Para finalizar aperta enter de novo.
  2. A opção "add untracked" permite adicionar novamente os arquivos que estão fora do stage
  3. Escolha a opção "quit" para sair

Ver status dos arquivos

git status Mostra quais alterações já foram para o stage

git log Mostra um histórico de commits. Para sair do log aperte "q"

Dando commit nas alterações

git commit -m "mensagem"

Alterando mensagem do commit anterior

git commit -m "mensagem" --amend

Desfazendo commit

git reset --soft HEAD~3 Desfaz os últimos 3 commits sem perder o que foi alterado

git reset --hard HEAD~3 Desfaz os últimos 3 commits perdendo tudo o que foi alterado neles

git rebase -i HEAD~4 Mostra os 4 últimos commits, possibilitado fazer alterações neles:

  1. É possível remover um commit e suas alterações mudando a palavra "pick" do commit para a letra "d"
  2. É possível mesclar vários commits mudando a segunda palavra "pick" por "s". Explicação nesse vídeo no minuto 6:15

Alterar editor de texto padrão do git

git config --global core.editor "nano"

Clonando repositório

git clone url_do_repositório Será criada uma pasta com o nome do repositório e seus arquivos dentro
