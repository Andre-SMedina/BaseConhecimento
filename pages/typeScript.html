Type Script

Vídeos

Hora de Codar

Instalar no node

npm i -g typescript

Criar arquivo de configuração do type script

tsc --init na pasta raiz do projeto

Definir tipo da variável em tsc

const x: number = 10; tipagem criada por annotation

const x = 10; tipagem criada por inferencia

As duas formas funcionam do mesmo jeito

const lista: number[] = [1, 2, 3]; tipagem de arrays

let tupla: [number, string, string[]; Exemplo: tupla = [5, "pedro", ["maria", "xuxa"]]

const objeto: {name: string, age: number} = {name: "andre", age: 40}; Tipagem de objetos

let qualquer: any = 0; Com a tipagem any, a variável pode receber quaquer tipo de valor. Exemplo: qualquer = "oi"

let multi: number | string = 0; Utilizando o pipe "|", é possível atribuir dois tipos possíveis a variável. Exemplo: multi = "oi"

type myType = number | string; Utilizando o "type", é possível armazenar na variável "myType" os tipos suportados. Exemplo: const x: myType = "paulo"; ou const z: myType = 30

enum Size {P = "Pequeno", M = "Médio", G = "Grande"}; "enum" serve para guardar valores que poderão ser utilzados posteriormente. Exemplo: const camisa = {name: "Camisa gola v", size: Size.G}

let literal: "autenticado" | null; O type literal não permite que o valor da variável seja alterado para um valor diferente do que já foi pré-definido. Neste exemplo, a variável literal só poderá receber o valor "autenticado" ou null. Exemplo: literal = "autenticado"

Tipando funções

Tipando os argumentos de uma função

function soma (a: number, b: number) {
          return a+b
        };

Tipando o retorno da função. Caso a função não retorne nada, deve-se usar ": void" onde define o retorno 

function hello (name: string): string {
            return `Hello ${name}`
          };

Para tornar um parâmetro opicional, utiliza-se o "?"

function saudacao(name: string, pronome?: string): void {
            if (pronome) {
              console.log(`Olá ${pronome} ${name}`)
            } else {
                console.log(`Olá ${name}`)
              }
          };

Para tornar um parâmetro genérico

function arrays<T>(arr: T[ ]) {
          arr.forEach((item) => {
            console.log(item)
          })  
        };
        const a = [1,2,3]
        const b = ["a","b","c"]
        arrays(a)
        arrays(b)

Classes

Definindo os parâmetros da classe

class User {
            name;
            age;
            admin;
            constructor(name: string, age: number, admin: boolean) {
              this.name = name;
              this.age = age;
              this.admin = admin;
            }
            showUserName(): void {
              console.log(`O nome do usuário é ${this.name}`)
            }
        }
        const pedro = new User("pedro", 30, false);

Criando interfaces

interface params {
          n1: number;
          n2: number;
        }
        function sum(nums: params): number {
          return nums.n1 + nums.n2
        }

Definir diretório root

No arquivo de configuração que foi criado a partir do comando "tsc --init", vá em modules, remova o comentário da linha "rootDir" e coloque o caminho da pasta src. "rootDir": "./src",

Definir diretório de saída

No arquivo de configuração "tsconfig.json", altere a linha "outDir" para direcionar para o diretório de saída (local onde vai ser gerado o arquivo.js após compilar o arquivo.ts com o comando "tsc" no terminal). Exemplo: "outDir": "./dist/js". Após isso, digitar no console o comando tsc e apertar enter

Compilação automática

Para o arquivo.ts ser compilado sempre que houver alteração, digite no terminal o comando "tsc -w"
